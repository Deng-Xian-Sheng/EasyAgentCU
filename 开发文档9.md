# EasyAgentCU 开发文档（第 9 部分：程序员任务拆分清单）

一份“可直接交给程序员的任务拆分清单”，按前端、后端、联调、测试逐项拆到开发事项级别。

下面这份清单按可执行开发任务拆开，拿去直接建项目 issue 就能开工。每个任务都写了产出物、关键实现点、完成标准、前置依赖。

---

## A. 项目初始化

### T01. 创建仓库基础结构

产出物：

`cmd/easyagentcu/main.go`  
`internal/...` 全部目录骨架  
`frontend/...` Next.js 工程  
`migrations/001_init.sql`  
`Makefile`  
`scripts/build-frontend.sh`  
`scripts/package.sh`

关键实现点：

按开发文档第 4 部分的目录一次性搭好。  
前端初始化 Next.js + React 19 + TypeScript + Tailwind。  
安装 shadcn/ui、radix-ui、prompt-kit、zustand、zod、sonner、lucide-react、noVNC。  
后端初始化 Go module，安装 chi、cors、zerolog、uuid、backoff、imaging、gg、gorilla/websocket、modernc sqlite。

完成标准：

仓库目录与文档一致。  
前后端都能本地启动空工程。  
`go build ./...` 通过。  
前端 `npm run build` 通过。

前置依赖：无。

## E. Agent 基础链路

### T15. EventEmitter 落库与广播

产出物：

`internal/agent/event_emitter.go`

关键实现点：

所有运行事件先写 `run_events` 再广播到 SSE。

完成标准：

能发出 `run.created`、`run.status.changed`、`run.phase_text.set`、`heartbeat` 等事件。  
数据库中有对应 `run_events` 记录。  
前端刷新后通过 runs 接口可恢复状态。

前置依赖：T10、T05。

### T16. ToolRegistry 与 ToolExecutor

`internal/agent/tool_registry.go`  
`internal/agent/tool_executor.go`

终止型工具 `finish_task`、`request_user_action` 单独处理。

完成标准：

非法坐标、非法 key 组合会返回 `ok=false` 的 tool result。
